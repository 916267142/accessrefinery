# The Correctness Experiment and Running AWS Access Analyzer

This repository provides a set of scripts to interact with **AWS Access Analyzer** and generate findings for S3 bucket policies.

## Main Scripts

- **bash.sh**: Contains the basic AWS CLI commands to update the bucket policy and trigger the resource scan.  
- **batch.sh**: Processes a directory of policies, runs AWS Access Analyzer, and generates findings for each policy file.  
- **compare.sh**: Compares the resulting intents from Access Analyzer and AccessMiner.

## Prerequisites for AWS Access Analyzer

1. Register for an AWS account.  
2. Create an S3 bucket in the AWS S3 service and configure a **resource-based** bucket policy.  
   > **Note**: Identity-based policies will not generate findings.  
   > Also, ensure that **Block all public access** is turned off for the bucket.

   ![alt text](/tools/bucket.png)  
   ![alt text](/tools/policy.png)

3. Follow the [AWS IAM Access Analyzer Samples GitHub repository](https://github.com/aws-samples/aws-iam-access-analyzer-samples) to configure the CLI. Install the AWS CLI with:

    ```bash
    sudo apt install awscli
    ```

4. Test the AWS CLI installation:

    ```bash
    aws help
    ```

5. Configure AWS credentials:

    ```bash
    aws configure
    ```

    Example output:

    ```
    AWS Access Key ID [None]: AKIATHVQKZH5ZHIHEGOH
    AWS Secret Access Key [None]: RptM4iuzr15LB1Z8gGEmLDX5whhs/yP/Xq+5SCpH
    Default region name [None]: ap-northeast-1
    Default output format [None]: json
    ```

6. Install `jq` for processing JSON files:

    ```bash
    sudo apt install jq
    ```

## Running AWS Access Analyzer

### 1. Prepare the Policy Directory

Make sure the directory (e.g., `data/Correctness`) contains all the S3 bucket policy `.json` files you want to analyze.

### 2. Run the Script

Execute the following command to process the policy files:

> **Note**:  
> - The script must be run from the project’s root directory.  
> - The input directory should be placed inside the `data/` folder.  
> - Results will be saved under `aws_result/Correctness/`.
> - 20 is the time limit : 20s, the process will end when exceeding this time.

```bash
sh tools/batch.sh data/Correctness 20
```
